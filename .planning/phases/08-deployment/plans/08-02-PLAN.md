08-02: Set Up Custom Domain in Vercel

## Goal
Configure custom domain r.alexdonega.com.br in Vercel for production access.

## Requirements
- DEPL-02: Custom domain r.alexdonega.com.br is configured and accessible

## Success Criteria
1. Custom domain is added in Vercel dashboard
2. SSL certificate is provisioned automatically
3. Domain points to Vercel's infrastructure
4. Site is accessible via custom domain

## Implementation Steps

### 1. Add Domain in Vercel
1. Go to Vercel Project → Settings → Domains
2. Enter domain: `r.alexdonega.com.br`
3. Click "Add"

### 2. Configure DNS
Vercel will provide DNS records to add:
- A record: `r` → `76.76.21.21` (Vercel IP)
- CNAME: (optional for www subdomain)

### 3. Wait for DNS Propagation
- Usually takes 5-30 minutes
- Can check with `dig r.alexdonega.com.br`

### 4. SSL Certificate
- Vercel automatically provisions Let's Encrypt certificate
- Certificate provisions once DNS is verified

### 5. Update Environment Variables
Add production domain to environment:
```
VITE_APP_URL=https://r.alexdonega.com.br
```

### 6. Configure Redirects (Optional)
```typescript
// vercel.json
{
  "redirects": [
    {
      "source": "/",
      "destination": "https://r.alexdonega.com.br",
      "has": [
        {
          "type": "host",
          "value": "your-vercel-app.vercel.app"
        }
      ]
    }
  ]
}
```

## Files to Modify
- `vercel.json` - Add redirects if needed
- Environment variables - Add VITE_APP_URL

## DNS Configuration

### Required Records
| Type | Name | Value |
|------|------|-------|
| A | r | 76.76.21.21 |
| A | r | 76.76.21.22 |

### Optional (for www)
| Type | Name | Value |
|------|------|-------|
| CNAME | www | cname.vercel-dns.com |

## Notes
- Vercel manages SSL automatically (Let's Encrypt)
- Domain must be verified before SSL is issued
- A records point to Vercel's edge network
- HTTPS is enabled by default
- Domain propagation can take up to 24 hours (usually faster)

## Verification
```bash
# Check DNS
dig r.alexdonega.com.br

# Check SSL
curl -I https://r.alexdonega.com.br

# Check redirect
curl -I http://r.alexdonega.com.br
```

## Troubleshooting
- DNS not propagating: Check with `dig` command
- SSL not provisioning: Ensure DNS is correct
- 404 errors: Check vercel.json rewrites
- Redirect loop: Check redirect configuration
